Defeito de Parametrização

 

Causas

Soluções

O parâmetro de entrada « modo de funcionamento » (Mode) não corresponde à um valor conhecido.

Verificar o valor da entrada Mode

- Mode = 1: funcionamento para perímetro flexível (com controle duplicado)

- Mode = 2: funcionamento para perímetro de escapamento

- Mode = 3: funcionamento para todos os perímetros (sem controle duplicado)

Os parâmetros de entrada TALO e BDD não estão dentro de uma localização correta ou não estão em um formato correto de dados.

Formatos de dados aceitos :

Tipo de dados

Descrição

STRUCT

Estruturas

ARRAY

Tabela

DB

Bloco de dados global

 

Zonas de memórias aceitas:

Zone

Description

DB

Bloco de dados

DI

Bloco de dados instanciados

O valor do parâmetro de entrada OffRef (localização de uma referência dentro de uma vaga) não está correto.

Verificar se o valor da entrada OffRef responde ao critério seguinte :

  - não é inferior a 0.

  - está compreendido entre 0 e a entrada LgEt - 1

O valor do parâmetro de entrada NbMaxEt (número máximo de vagas) não está correto.

O valor da entrada NbMaxEt não deve ser inferior a 2 vagas

Para as funções de leitura, modificação e exclusão:

 

Os parâmetros de entrada OffEt e Nmot  não correspondem à dimensão máxima de uma vaga (LgEt)

Verificar se:

- a entrada OffEt é superior ou igual a 0 e inferior a dimensão máxima de uma vaga (LgEt)

- a entrada Nmot é inferior à dimensão máxima de uma vaga (LgEt)

- o calculo OffEt + Nmot não excede a dimensão máxima de uma vaga (LgEt)

A dimensão da tabela local (TALO) não corresponde ao parâmetro de entrada LgEt (tamanho de uma vaga).

Verificar se:

  - a tabela local (TALO) possui uma dimensão superior ou igual ao parâmetro de entrada LgEt

A base de dados (BDD) não pode acumular o número de dados desejados.

Verificar se:

  - a base de dados (BDD) possui uma dimensão superior ou igual ao calculo NbMaxEt x LgEt
