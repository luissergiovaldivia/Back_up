获取工作台转移故障

 

原因

故障

TAB_ECH 至 TAB_ACQ转移故障：IhmInfo = 81

 

“iptol交换工作台”至“自动机获取工作台”的转移中出现故障

 

只有在siptol建议一个零件并且调试模式开启时故障才会出现

 

检查如下的参数TAB_ECH 和 TAB_ACQ：

 

- 存储区域不得重叠          

- 交织深度超多（见文件西门子SFC20 ‘’BLKMOV’’)

- 数据位于无法使用的数据块中（写入保护，无法实行或者无法装载）

- 如果参数的类型为BOOL，所指长度应为8的倍数

TAB_MAP 至 TAB_ACQ转移故障（第一部分）：IhmInfo = 82

“调试工作台”至“自动机获取工作台”的转移中出现故障

 

 

只有在调试模式被选择而且操作员下达启动指令时故障才会出现

 

检查如下的参数TAB_MAP 和 TAB_ACQ：

 

- 存储区域不得重叠          

- 交织深度超多（见文件西门子SFC20 ‘’BLKMOV’’)

- 数据位于无法使用的数据块中（写入保护，无法实行或者无法装载）

- 如果参数的类型为BOOL，所指长度应为8的倍数

TAB_MAP 至 TAB_ACQ转移故障（第二部分）：IhmInfo = 83

“调试工作台”至“自动机获取工作台”的转移中出现故障（只有TABTyp = 7时）

 

 

 

只有在调试模式被选择而且操作员下达启动指令时故障才会出现

 

 

检查如下的参数TAB_MAP 和 TAB_ACQ :

 

- 存储区域不得重叠          

- 交织深度超多（见文件西门子SFC20 ‘’BLKMOV’’)

- 数据位于无法使用的数据块中（写入保护，无法实行或者无法装载）

- 如果参数的类型为BOOL，所指长度应为8的倍数

 

TAB_MAPEx 至 TAB_ACQ转移故障：IhmInfo = 84

 

“调试工作台”至“自动机获取工作台”的转移中出现故障（只有TABTyp = 10时）

 

只有在调试模式被选择而且操作员下达启动指令时故障才会出现

程序定义至TAB_ACQ转移故障：IhmInfo = 85

当程序定义的拷贝被屏幕页面查询时故障出现（只有在LTyp =3或4时）

 

只有在调试模式被选择而且操作员下达托盘启动指令时故障才会出现

检查如下参数TAB_ACQ :

- 数据位于无法使用的数据块中（写入保护，无法实行或者无法装载）
