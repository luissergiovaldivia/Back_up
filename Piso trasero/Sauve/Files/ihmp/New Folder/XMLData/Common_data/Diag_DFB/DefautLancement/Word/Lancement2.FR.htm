Dialogue incorrect : IhmInfo = 17

La valeur des données ‘’valcal’’ et ‘’valapi’’ du bloc d’échange ne correspondent pas à un dialogue connu.

Liste des dialogues possible : VALCAL / VALAPI

VALCAL = 1  / VALAPI = 0 : Le calculateur propose une pièce

VALCAL = 2  / VALAPI = 0 : Le calculateur ne propose pas de pièce

VALCAL = 0  / VALAPI = 2 : L’automate répond et attend une proposition du calculateur

VALCAL = 0  / VALAPI = 1 : L’automate a fait la pièce

 

1) Vérifier la valeur des mots ‘’VALCAL’’ et ‘’VALAPI’’ de la table d’échange

2) Si ‘’VALCAL’’ non conforme : Appeler l’opérateur siptol pour correction (valeur non pris en charge sur point de lancement)

3) Si ‘’VALAPI’’ non conforme : Erreur de programmation BF_LANCEMENT (modification du code programme)

 

Dialogue non initialisé : IhmInfo = 18

Les données ‘’valcal’’ et ‘’valapi’’ du bloc d’échange sont à zéro.

1) A l’aide d’une table d’animation renseigner les valeurs suivantes dans la table d’échange siptol :

                VALCAL = 0    VALAPI = 1    ETFLUX = 0

2) Appeler l’opérateur siptol pour acquitter l’alarme sur l’écran IHM siptol.
