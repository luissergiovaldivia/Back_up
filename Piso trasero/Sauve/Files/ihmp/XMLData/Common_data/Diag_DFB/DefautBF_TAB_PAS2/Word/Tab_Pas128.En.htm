Explanation of the defect

Cause(s)/ Solution(s)

MEMORY INCONSISTENCY DEFECT OF THE SUPPORT / PHYSICAL CONDITION:

DIAG.CodeDef = 128

 

- the condition of the support presence memory is inconsistent with the physical condition indicated by the sensor(s) which are fitted to the handling element.

 

See the IHMINFO value to find the solution:

 

1 – The DPA  sensor is in an inconsistent condition with the support presence memory.

 

2 – The DPVA  sensor is in an inconsistent condition with the support presence memory.

 

3 – The DDPA  sensor is in an inconsistent condition with the support presence memory.

 

4 – The DPR  sensor is in an inconsistent condition with the support presence memory.

 

5 – The DPVR  sensor is in an inconsistent condition with the support presence memory.

 

6 – The DDPR  sensor is in an inconsistent condition with the support presence memory.

 

9 – The DPA and/or DPR sensors are inconsistent with the support presence memory.

 

Check the sensor(s). Check the condition of the support if it is present.

Check the condition of the support presence memory.

Upload the presence memory to the operation/ maintenance page if required.

Intervention and recycling procedure

Ensure safety AND Shift to manual mode.

Solve the cause(s) of the defect.

-       If the cause(s) of the defect are solved, IHMINFO is set to 0.

-       Press the BADI defect(s) cancellation pushbutton.

-       Turn the selector to AUTO, press the Start cycle pushbutton.
