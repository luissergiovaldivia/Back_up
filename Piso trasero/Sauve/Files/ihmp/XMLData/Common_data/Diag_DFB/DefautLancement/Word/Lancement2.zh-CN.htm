原因

故障

对话不正确：IhmInfo = 17

交换块的‘’valcal’’ 和 ‘’valapi’’数据值同已知的对话不符

可能的对话清单：VALCAL / VALAPI

VALCAL = 1  / VALAPI = 0 : 计算器建议一个零件

VALCAL = 2  / VALAPI = 0 : 计算器不建议零件

VALCAL = 0  / VALAPI = 2 : 自动机回答并等待计算器的建议

VALCAL = 0  / VALAPI = 1 : 自动机做了零件

 

1) 

检查交换工作台的‘’VALCAL’’ 和 ‘’VALAPI’’字值

2) 

如果‘’VALCAL’’不合格：致电siptol操作员予以纠正（在启动点该值不被承担）

3) 

如果‘’VALAPI’’不合格：BF_LANCEMENT程序错误（程序编码修改）

对话未初始化：IhmInfo = 18

 

交换块的‘’valcal’’ 和 ‘’valapi’’数据为0

1)    借助动画工作台查询siptol交换工作台的如下数值：

VALCAL = 0    VALAPI = 1    ETFLUX = 0

2)     致电siptol操作员以便排除siptol IHM屏幕上的警报
