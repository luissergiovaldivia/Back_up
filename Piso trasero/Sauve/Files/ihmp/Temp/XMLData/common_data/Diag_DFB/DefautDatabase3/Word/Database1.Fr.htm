Défaut de paramétrage

 

Causes

Remèdes

Le paramètre d’entrée mode de fonctionnement (Mode) ne correspond pas à une valeur connue.

Vérifier la valeur de l’entrée Mode

- Mode = 1 : fonctionnement pour périmètre flexible (avec contrôle doublon)

- Mode = 2 : fonctionnement pour périmètre échappement

- Mode = 3 : fonctionnement tout périmètre (sans contrôle doublon)

Les paramètres d’entrées TALO et BDD ne sont pas dans un emplacement correct ou n’ont pas le bon format de données.

Formats de données acceptées :

Type de données

Description

STRUCT

Structures

ARRAY

Bloc de données global

DB

Bloc de données

Bloc de données d’instance

La valeur du paramètre d’entrée OffRef (emplacement de la référence dans un emplacement) n’est pas correcte.

Vérifier que la valeur de l’entrée OffRef répond aux critères suivant :

  - n’est pas inférieure à 0

  - est compris entre 0 et l’entrée LgEt -1

La valeur du paramètre d’entré NbMaxEt (nombre maximum d’étiquette) n’est pas correcte.

La valeur de l’entrée NbMaxEt ne doit pas être inférieure à 2 emplacements

Pour les fonctions lecture, modification et effacement :

 

Les paramètres d’entrées OffEt et Nmot ne correspondent pas à la dimension maximum d’une étiquette (LgEt).

Vérifier que :

- l’entrée OffEt est supérieure ou égal à 0 et inferieure à la dimension maxi d’une étiquette (LgEt)

- l’entrée Nmot est inférieure à la dimension maxi d’une étiquette (LgEt)

- le calcule OffEt + Nmot ne dépassent pas la dimension maxi d’une étiquette (LgEt)

La dimension de la table locale (TALO) ne correspond pas au paramètre d’entrée LgEt (longueur d’une étiquette)

Vérifier que :

  - la table locale (TALO) a une dimension supérieure ou égal au paramètre d’entrée LgEt

La base de données (BDD) ne peut accueillir le nombre de données souhaité

Vérifier que :

  - La base de données (BDD) a une dimension supérieure ou égale au calcul NbMaxEt x LgEt
