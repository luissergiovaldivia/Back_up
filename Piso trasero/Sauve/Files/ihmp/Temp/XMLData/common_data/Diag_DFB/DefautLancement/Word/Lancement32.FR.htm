Causes

  Défaut transfert table d’acquisition

 

Causes

Remèdes

Défaut transfert TAB_ECH vers TAB_ACQ : IhmInfo = 81

Un défaut est survenu lors du transfert de la ‘’Table d’échange siptol’’ vers la ‘’Table d’acquisition automate’’.

 

Ce défaut ne peut apparaître que lorsque siptol propose une pièce et que le mode mise au point n’est pas actif.

 

Vérifiez les points suivants pour les paramètres TAB_ECH et TAB_ACQ :

 

- Les zones mémoires ne doivent pas se chevaucher.           

- Profondeur d'imbrication dépassée (voir fiche siemens SFC20 ‘’BLKMOV’’).

- Données situées dans un bloc de données non utilisable (protégé en écriture, non exécutable ou non chargé).

- Si les paramètres sont de type BOOL, la longueur indiquée doit être un multiple de 8.

Vérifiez les points suivants pour les paramètres TAB_MAP et TAB_ACQ :
