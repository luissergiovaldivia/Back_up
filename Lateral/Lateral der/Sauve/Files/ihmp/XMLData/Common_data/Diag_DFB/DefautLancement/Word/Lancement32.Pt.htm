Defeiro de transferência da tabela de aquisição

 

Causas

Soluções

Defeito transferência TAB_ECH para TAB_ACQ : IhmInfo = 81

Um defeito surgiu durante a transferência da ‘’Tabela de troca SIPTOL’’ para ‘’Tabela de aquisição do CLP’’.

 

Este defeito aparece apenas quando o SIPTOL propõem uma peça e/ou quando o modo MAP está ativo.

 

Verificar os pontos seguintes para os parâmetros TAB_ECH e TAB_ACQ :

 

- As zonas de memórias não devem se sobrepor.     

- Profundidade de nidificação ultrapassada (ver o manual siemens para SFC20 ‘’BLKMOV’’).

- Dados situados dentro de um bloco de dados não disponível (protegido contra escrita, não executável ou sem espaço).

- Se os parâmetros são do tipo BOOL, o tamanho indicado deve ser um múltiplo.
